# 非角色机制

## 设计原则

非角色机制不是新的 Agent 角色，也不拥有独立身份、日志或长期职责。它们是当前会话 Agent 在特定条件下执行的一组检查清单。

人类用户是项目 Owner 和实际项目经理，负责决定是否启动、是否接受结果、是否继续推进。Agent 负责按清单执行、指出问题、记录证据和等待用户确认。

## 机制总表

| 机制 | 触发时机 | 谁启动 | 谁执行 | 结束条件 |
|------|----------|--------|--------|----------|
| Bootstrap 初始化 | 空项目、缺少入口文件、缺少项目上下文或进度索引 | 用户明确要求，或 Agent 检测到缺失后建议 | 当前会话 Agent | `CLAUDE.md`、项目上下文、进度目录、角色日志和初始迭代记录已创建，并由用户确认 |
| 迭代关闭检查 | DevOps（运维/部署工程师）部署通过、部署跳过且用户接受、或用户要求关闭迭代 | 用户明确要求，或 DevOps（运维/部署工程师）/Tester（测试工程师）/当前 Agent 建议 | 当前会话 Agent | `INDEX.md`、迭代记录、Review 摘要、部署状态、遗留项一致，并由用户确认关闭 |
| 流程审计 | 状态混乱、角色冲突、门禁不一致、用户质疑流程、或连续两轮以上卡住 | 用户明确要求，或任一角色在日志中提出 `[基线修正提案]` | 当前会话 Agent，必要时按 Role Creator（角色创建者）流程起草修改提案 | 输出问题清单、建议修正、是否需要修改基线；基线修改仍需用户确认 |

## 1. Bootstrap 初始化

### 启动口令

```text
执行 Bootstrap 初始化流程。
```

### 自动建议触发

如果 Agent 启动时发现以下任一情况，应停止常规角色工作，并建议用户先执行 Bootstrap：

- 缺少 `CLAUDE.md`
- 缺少 `docs/baseline/project-context.md`
- 缺少 `docs/progress/INDEX.md`
- 缺少 `docs/progress/iterations/`
- 缺少 `docs/progress/roles/`

### 执行者

当前会话 Agent 执行。它不变成 Project Manager，也不创建项目经理日志。执行前必须说明：Bootstrap 是一次性初始化机制，不是常驻角色。

### 输出

- 项目入口和上下文文件
- 初始进度索引
- 角色日志和纠错记录
- 初始迭代记录
- 下一步建议：通常是进入 PM 的 PRD 阶段

## 2. 迭代关闭检查

### 启动口令

```text
执行迭代关闭检查。
```

### 自动建议触发

如果当前 Agent 发现以下任一情况，应建议执行迭代关闭检查：

- 测试阶段已通过，且部署检查已通过
- 部署检查被标记为已跳过，且原因已记录
- 当前迭代所有阶段均为 `已定稿` 或 `已跳过`
- 用户问“这个迭代是不是结束了”

### 执行者

当前会话 Agent 执行。若用户刚以 DevOps（运维/部署工程师）或 Tester（测试工程师）角色工作，可以由该角色顺手执行关闭检查；若用户单独要求关闭检查，则不用切换角色。

### 检查项

1. `docs/progress/iterations/vX.Y.md` 中各阶段状态一致。
2. PRD、UI 方案、设计文档、实现 Review、测试报告、部署检查均有结论。
3. 所有阻塞项已关闭，或明确延期并得到用户接受。
4. `docs/progress/INDEX.md` 的当前迭代状态已更新。
5. 角色日志中没有未处理的关键阻塞。

### 输出

- 关闭结论：可关闭 / 不可关闭 / 有条件关闭
- 未关闭原因或延期项
- 用户确认后，更新迭代记录和 `INDEX.md`

## 3. 流程审计

### 启动口令

```text
执行流程审计。
```

### 自动建议触发

如果出现以下情况，Agent 应建议流程审计：

- 阶段状态互相矛盾
- Review 轮次对不上
- 某角色越权修改产出物
- 连续两轮 Review 卡在同类问题上
- 用户质疑工作流设计是否合理
- 角色日志出现 `[基线修正提案]`

### 执行者

当前会话 Agent 执行初步审计。若审计结论涉及新增、删除或修改角色，应切换到 Role Creator（角色创建者）流程起草提案。任何 `baseline/` 修改都必须由用户确认。

### 输出

- 问题清单
- 严重程度
- 建议修正
- 是否需要基线修正提案
- 若需要修改基线，列出拟改文件和理由，等待用户确认
